# Roadmap

这份 Roadmap 面向公开 GitHub 用户，用来说明 WorkspaceKit 的整体方向，不记录本机路径、测试包、主包等内部开发信息。

## 已完成

- 修复 Canvas Groups 包装 `loadGraphData()` 时改变零参数调用、导致多插件环境恢复上次工作流读取 `extra` 失败的问题。
- 统一 WorkspaceKit 侧边栏入口，包含工作流、节点、模板三个标签。
- Workflows2 工作流文件夹管理、拖拽整理、排序、最近工作流、插件回收站和系统回收站交接。
- Nodes2 节点浏览、搜索、收藏、收藏分组、官方收藏导入 / 导出和大节点库缓存。
- Templates 模板功能，可把选中的已连接节点保存为可复用模板。
- 模板分组、子分组、搜索、排序、拖拽整理、预览和行内删除确认。
- Templates 首次打开路径：空闲预取、同页面会话请求复用、分段耗时记录，并在 Templates 活跃时延后过期节点定义刷新。
- Canvas Groups 编组增强，包括 WorkspaceKit 编组快捷键和标题栏样式设置。
- 编组标题栏动作：执行当前编组输出节点、忽略、禁用，并逐节点保存 / 恢复原执行模式。
- Title2 视觉标题 / 注释节点。
- WorkspaceKit 设置面板第一版。
- 中英文本地化基础设施，以及“编组设置 / 透明标题”弹窗的双语改造。
- 持久化服务端 `/object_info` 快照缓存：签名校验、压缩分块上传、原子磁盘写入和浏览器 IndexedDB 复用。
- 英文和中文 README、Issue 模板、PR 模板、贡献说明和安全策略。

## 稳定性修复批次

以下批次根据大规模插件环境和主包体验反馈排序。每批独立验证，确认结果后再进入下一批，避免同时修改过多功能而无法定位回归。

1. **工作流重复同步止损（进行中）**
   - 普通打开优先复用 ComfyUI 官方工作流 Store，目标不存在时才同步。
   - 避免新建工作流在打开过程中重复同步官方列表。
   - 验收：普通打开不触发 `syncWorkflows()`；新建最多同步一次。
2. **性能基线与节点缓存协调（进行中）**
   - 为启动、工作流扫描、节点缓存、`/object_info`、模板请求和渲染增加分段耗时记录。
   - 使用服务端共享缓存和浏览器 IndexedDB 快照，页面刷新及多标签页共用单一后台更新任务。
   - 未进入 Nodes2 时不主动构建完整节点索引；插件变化检查采用低优先级、分批执行。
   - 验收：缓存命中时 Nodes2 首屏目标小于 500ms；多标签页不重复构建索引。
   - 当前：前端分段耗时、轻量节点签名、浏览器 IndexedDB 复用、跨标签页单任务协调和签名保护的服务端快照缓存均已实现。2026-07-16，旧快照在运行环境从 198 个插件 / 6,135 节点变为 200 个插件 / 6,323 节点时被正确拒绝；打开 Nodes2 后已重建并原子写入匹配快照。有效快照现通过协商 gzip 返回，6,323 节点快照实际传输 11.89 MB，而未压缩约为 50.35 MB。多标签耗时测量仍是发布验收项，不再是实现阻塞。
3. **Templates 延迟复测与解耦**
   - 区分接口、JSON 解析、预览和浏览器主线程阻塞耗时。
   - 节点后台更新不得阻塞 Templates 首屏；大型模板按需生成预览。
   - 当前：已在测试包的 2 个模板环境完成验证：空闲预取请求 128.1ms、数据整理 0.2ms、首次渲染 2.6ms；Templates 保持活跃期间未触发 `/object_info` 刷新，用户体验验收已通过。主包复测仍是发布门槛。
4. **Workflows2 增量更新**
   - 新建、重命名、移动、删除和恢复先更新本地状态，再在后台合并刷新。
   - 移除操作后的全目录重复扫描和阻塞式官方同步。
   - 建立重命名、移动、删除、恢复、保存和官方列表同步回归清单。
   - 当前：已在 `docs/TESTING.md` 建立 P0 回归清单，待完整执行。已合并操作后的官方列表刷新；新建文件夹 / 工作流、重命名、移动、移入回收站、恢复和清理回收站已使用本地增量列表。切换根目录因数据源整体变化，保留一次必要的完整加载。
5. **面板信息架构与玻璃效果**
   - 将“最近工作流”改为“打开”，增加“浏览”区域；两区支持折叠并保存状态。
   - 使用标题层级、留白和标题延伸线区分区域，避免大面积色块和重复分割线。
   - 修复玻璃背景层级、透明度与模糊滑块联动。
   - 当前：“打开”/“浏览”已支持折叠并保存状态；节点一级分区也已支持折叠，并共用分区标题结构。工作流两区已改为共同内容容器的相邻分区，顶部容器不再位于两区之间，因此不再引入顶部底边框；节点分区保持现状。背景设置已改为“透明背景 / 磨砂玻璃”互斥单选；毛玻璃使用固定模糊、雾面层和5–95透明度控制。测试包体验验收已通过，发布前保留一次主包视觉回归。
6. **工程化与发布准备**
   - 继续拆分 `entry.js`，至少拆出 workflows、nodes、templates 和 settings。
   - 加入 Python、JavaScript、JSON 和服务层测试及最小 CI。
   - 增加插件数据导出 / 导入、schema version 和导入前自动备份。
   - 补齐截图、GIF 教程和 Registry 视觉品牌资料（图标与 Banner）。
- 当前：已抽出 API、常量、性能记录、节点缓存协调、节点面板本地状态、节点库归一化模块、节点库首次加载模块、节点对象信息状态模块、Templates 库数据层、官方工作流适配器、工作流“打开记录”存储模块、“当前工作流状态”模块、“工作流项数据”模块、“工作流路径状态”模块、“打开 / 浏览”分区外壳、“工作流树构建”模块、“只读浏览搜索”模块、“浏览结果刷新”生命周期、“浏览右键菜单”渲染器、“回收站列表”渲染器、“浏览行”渲染器，以及“排序菜单”渲染器。Issue / PR 模板、贡献说明、安全策略和 Registry 发布门控已完成；WorkspaceKit 0.2.2 已发布到 Comfy Registry。“排序菜单”的打开与排序操作已完成 UI 验收；仍需在一次正常加载的测试包页面验收 Esc 关闭。下一步按“剩余节点缓存生命周期 → 节点 UI → Templates UI → 设置”的顺序继续拆分，每批保留回归验证。

测试过程、性能数据与尚未确认根因的错误记录在 `docs/TESTING.md`。
